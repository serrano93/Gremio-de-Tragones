fix: bypass Supabase JS client entirely for user/merchant flow

Root cause: @supabase/supabase-js v2 has internal _recoverAndRefresh
and _initialize methods that block on slow networks. Every query
(getSession, fetchProfile) was hanging in this recovery loop, causing:
- Infinite loading spinners
- SIGNED_IN events firing repeatedly
- App stuck in isLoading=true

Fix: bypass the Supabase JS client completely for user-facing flows.
- supabase.ts: persistSession=false, autoRefreshToken=false, no PKCE
- New restQuery() and restRpc() helpers using fetch + AbortController
- useAuth: reads session from localStorage directly, no getSession
- useMissions/useOffers/useEstablishments: use restQuery with 8s timeout
- ScanPage: uses restRpc for mission/offer validation
- ProfilePage: login/register with fetch direct, reload on success
- AppRouter: removed AuthGate/LoadingScreen, app shows immediately

The Supabase JS client is still imported by admin pages but those
are not in the critical path. Auto-refresh disabled so it never
hangs.
